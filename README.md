# 🧠 The (In)frequent Brain 2.0

> "I rarely use my own brain, but here’s another one that I might use just as rarely."

Welcome to my **Second Brain**. This is a digital sanctuary for knowledge, research, and project planning—a place where ideas come to live (and occasionally gather digital dust).

## 🚀 Overview
This vault is a structured repository built using the **PARA Method** (Projects, Areas, Resources, Archives). It serves as my primary hub for deep technical research, academic study, and long-term project building.

## 📂 Architecture
The vault is organized into a modular system to ensure that information is easy to find, even if I forget I wrote it:

- **`00 - Inbox`**: The digital waiting room for unprocessed thoughts.
- **`10 - Projects`**: Active, goal-oriented endeavors with defined deadlines.
- **`20 - Areas`**: Ongoing technical domains including:
  - **Cyber Security**: Fundamentals, Network Security, and Offensive Security.
  - **Reverse Engineering**: Low-level analysis and binary exploration.
  - **DevOps**: Infrastructure automation and CI/CD pipelines.
  - **Mobile Development**: Dart & Flutter Bootcamp materials.
- **`30 - Resources`**: A library of interests, reference materials, and useful snippets.
- **`40 - Archive`**: The graveyard of completed projects and outdated curiosities.
- **`50 - System`**: The engine room (Templates, Attachments, and Daily Logs).

## 🛠️ Tech Stack
This brain is powered by:
- **Obsidian**: The core knowledge graph.
- **Templater**: For consistent, automated note structures.
- **Dataview**: For dynamic querying across the vault.
- **Tasks**: For managing the "to-do" list my real brain ignores.
- **Git**: For version control (with a very strict `.gitignore` for secrets).

## 🔒 Privacy & Security
As this vault contains sensitive research and technical data, it is protected by a robust security policy. 
- **No Secrets**: API keys, credentials, and private tokens are excluded via `.gitignore`.
- **Local First**: Data remains local and encrypted where necessary.
