## ☁️ Cloud Architect Learning Path

### Phase 1: Foundations
1. **01 - Cloud Fundamentals**
	- [ ] NIST Definition: 5 Characteristics, 3 Service Models (IaaS, PaaS, SaaS), 4 Deployment Models (Public, Private, Hybrid, Community).
	- [ ] Shared Responsibility Model (Customer vs. Provider).
	- [ ] Economics: CapEx vs. OpEx, Pay-as-you-go.

### Phase 2: Compute & Virtualization
2. **02 - Virtualization & Computing**
	- [ ] Virtual Machines (AWS EC2, Azure VM, GCP Compute Engine).
	- [ ] Auto-scaling & Load Balancing (ELB/ALB/NLB).
	- [ ] Vertical vs. Horizontal Scaling.
3. **03 - Storage & Databases**
	- [ ] Object Storage (S3, Blob Storage, Cloud Storage).
	- [ ] Block Storage (EBS, Azure Managed Disks).
	- [ ] Relational (RDS, SQL Database) vs. NoSQL (DynamoDB, CosmosDB).

### Phase 3: Infrastructure & Networking
4. **04 - Networking & Content Delivery**
	- [ ] Virtual Private Clouds (VPC), Subnets, Route Tables.
	- [ ] DNS Management (Route 53, Azure DNS).
	- [ ] Content Delivery Networks (CloudFront, Cloud CDN).
5. **05 - Security & Identity**
	- [ ] Identity & Access Management (IAM): Users, Roles, Policies.
	- [ ] Key Management Service (KMS) & Encryption.
	- [ ] Web Application Firewalls (WAF) & DDoS Protection (Shield).

### Phase 4: Advanced Operations & Serverless
6. **06 - Monitoring & Operations**
	- [ ] Monitoring & Logging (CloudWatch, CloudTrail, Azure Monitor).
	- [ ] Infrastructure as Code (CloudFormation, ARM Templates, Terraform).
7. **07 - Serverless & Microservices**
	- [ ] Function as a Service (AWS Lambda, Azure Functions, Google Cloud Functions).
	- [ ] Event-driven Architectures.
	- [ ] Containers (ECS, EKS, GKE, Fargate).
8. **08 - Multi-Cloud & Hybrid Strategy**
	- [ ] Connecting On-Prem to Cloud (Direct Connect, VPN).
	- [ ] Comparing AWS vs. Azure vs. GCP services.

## 🏆 Bootcamp Projects
- [ ] **Static Website**: Host a portfolio using S3 and CloudFront with SSL.
- [ ] **Highly Available Web App**: Multi-AZ deployment with ALB and Auto-scaling.
- [ ] **Serverless API**: Lambda + API Gateway + DynamoDB.
