# Study Note: Introduction to Web2 & Web3
Subject: #study/webdev #study/web3 
Status: #study/learning 

## 📖 Key Concepts & Definitions
- **Web1 (Static Web)**: "Read-only." HTML pages with little interaction.
- **Web2 (Social Web)**: "Read-write." Characterized by user-generated content, centralized platforms (Facebook, Google), and data monetization.
- **Web3 (Decentralized Web)**: "Read-write-own." Built on blockchain technology, focusing on decentralization, user ownership, and peer-to-peer interaction.
- **DApp (Decentralized Application)**: An application that runs on a decentralized network (like Ethereum) rather than a centralized server.

## 📝 Detailed Notes

### The Core Pillars of Web3
1. **Decentralization**: No single entity controls the network.
2. **Permissionless**: Anyone can participate in the network.
3. **Trustless**: You don't need to trust a third party (like a bank); you trust the code (Smart Contracts).
4. **Blockchain**: A distributed ledger that records all transactions.

### Web2 vs. Web3 Comparison
| Feature | Web2 | Web3 |
| ------- | ---- | ---- |
| **Control** | Centralized (Companies) | Decentralized (Users/Nodes) |
| **Data Storage** | Centralized Servers | Distributed Ledger (Blockchain) |
| **Privacy** | Often Monetized | User-controlled (Pseudonymous) |
| **Trust** | Third-party (Intermediaries) | Code/Consensus |

## 🧠 Questions & Flashcards
- **Q**: What is a "Smart Contract"?
  - **A**: A self-executing contract with the terms of the agreement directly written into code.
- **Q**: How do users interact with a Web3 application?
  - **A**: Usually through a **Wallet** (like Metamask) which stores their private keys and identity.
